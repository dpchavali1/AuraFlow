## BUILD & TOOLCHAIN MISTAKES

<!-- Gradle, AGP, SDK, dependency conflicts -->

| ID | Date | Phase | Problem | Root Cause | Fix | Recurrence Risk |
|----|------|-------|---------|------------|-----|-----------------|
| B-001 | 2026-04-02 | 1 | `Ktor 3.4.2` fails with "Module was compiled with an incompatible version of Kotlin. Binary metadata 2.3.0, expected 2.1.0" | Build prompt locked Kotlin at 2.1.20 but Ktor 3.4.2 requires Kotlin 2.3.x. Kotlin binary metadata is NOT forwards-compatible. | Upgrade Kotlin from 2.1.20 to 2.3.10. Update KSP accordingly. Also update CMP if needed. | Any time Ktor or CMP versions are bumped ŌĆö check Kotlin binary metadata compatibility |
| B-002 | 2026-04-03 | 13 | `kotlinx.datetime.Clock.System` unresolved reference | In kotlinx-datetime 0.7.x, `Clock` was aligned with `kotlin.time.Clock`. The `kotlinx.datetime.Clock` import causes `System` to be unresolved. | Use `import kotlin.time.Clock` instead of `import kotlinx.datetime.Clock`. Both `todayIn()` and `now()` work on `kotlin.time.Clock.System`. | Every time kotlinx-datetime is used ŌĆö always import from `kotlin.time.Clock` not `kotlinx.datetime.Clock` |
| B-003 | 2026-04-03 | 13 | `AuraFlowDatabaseConstructor` not abstract and missing `initialize()` | Room KMP `expect object` needs to declare `override fun initialize(): AuraFlowDatabase` even though the actual is generated by KSP. | Add `override fun initialize(): AuraFlowDatabase` to the expect object body. | Adding new Room database entities ŌĆö always check expect constructor declaration |
| B-004 | 2026-04-03 | all | Stale Gradle daemon lock file `registry.bin.lock` blocks ALL builds including --no-daemon | Background Bash commands started multiple Gradle daemons simultaneously. One crashed holding the registry lock. Gradle's 2-minute timeout does not release file system locks when the holding process dies without cleanup. | User must manually `rm ~/.gradle/daemon/9.4.1/registry.bin.lock`. Do NOT start multiple background Gradle builds simultaneously ŌĆö always wait for one to finish before starting another. | HIGH: Do not use `run_in_background` for multiple Gradle builds in parallel |
| B-005 | 2026-04-03 | 13 | `AchievementSystem` constructor had `PlayerRepository` param that was never used | Koin DI requires `AchievementSystem(get(), get())` but repository wasn't called internally, creating unnecessary database coupling in tests. | Removed `PlayerRepository` from `AchievementSystem` constructor. It only uses `Settings`. | When designing a class that only needs Settings, don't inject unused repository params |
